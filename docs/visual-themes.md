# 文山视觉皮肤

## 先判断当前曲线为什么“差点意思”

当前 `survey-parchment` 已经解决了孤立圆环问题：山峰由一个全局高度场连接，山间关系会形成山脊和鞍部。

但曲线仍然容易带有“数学等值线”的感觉，原因不是颜色，而是地形生成还偏向：

```text
椭圆峰体
+ 平滑关系脊
+ marching squares
```

它缺少真实山脉常见的：

- 多尺度主脊与支脊；
- 局部侵蚀形成的凹口；
- 非均匀坡度；
- 同一条等高线上的缓急变化；
- 河谷式负空间；
- 手工制图的线条呼吸。

继续加阴影或纸张噪点只会让页面变脏，不能解决地形结构问题。

---

## 所有皮肤共享的底线

- 等高线是核心视觉资产，不能移除。
- 地图必须是一张山脉群，不是若干孤立圆环。
- 山名、篇数和证据点必须清楚。
- 海拔由独立 canonical 文章数决定。
- 山间位置由审阅关系决定。
- 单色优先；强调色不得承担必要语义。
- 不使用模糊地形阴影。
- 纸张纹理必须低于 2% 可见度，不能覆盖等高线。
- 不复制现成电影、游戏或文学 IP 的地图字体、图标和构图。

---

## 皮肤 1：survey-parchment

当前默认。

### 关键词

```text
精密测绘
羊皮纸
现代档案
黑白灰
仪器控制台
```

### 特征

- 冷灰羊皮纸；
- 规则网格；
- 主次等高线；
- 无填色；
- 外置控制台；
- 适合分析报告和长期知识资产。

### 下一步曲线升级

不要增加装饰，改地形算法：

1. 先从山间关系图生成主山脊骨架；
2. 沿主山脊生成二级支脊；
3. 使用带限分形噪声扰动高度场，而不是直接抖动线条；
4. 用负高度场切出少量河谷；
5. 每三至四条绘制一条 index contour；
6. 让线宽随海拔和证据量变化，但保持单色。

## 皮肤 2：mythic-parchment

推荐作为第二套皮肤。

它可以吸收“中土世界羊皮纸地图”的情绪：古老、文学、旅程、未知疆域。但不复制 Tolkien 地图的字体、山形符号、边框或具体构图。

### 关键词

```text
古代幻想制图
旅行手稿
刻线山脉
温暖羊皮纸
文学地图
```

### 视觉结构

- 背景：温暖的未漂白纸色，而不是黄色滤镜；
- 山脉：等高线仍为主体，局部增加沿坡向的短 hachure；
- 线条：深褐黑，不使用纯黑大面积填充；
- 标题：现代可读衬线体，不使用仿 Tolkien 字体；
- 边框：极简地图框，不加卷轴、火漆、羽毛笔等陈词滥调；
- 选中态：一圈手绘感测量圆，不用彩色光晕；
- 文章抽屉：像田野笔记卡，而不是游戏任务面板。

### 地形算法

```text
审阅关系图
→ 主脊最小生成骨架
→ 强关系增加并行支脊
→ 峰值高度场
→ 低频地质折叠
→ 中频侵蚀凹口
→ 少量高频带限扰动
→ 全局等高线
→ 局部坡线
```

关键点：崎岖来自高度场，不来自给最终曲线加随机毛刺。

## 皮肤 3：archive-engraving

### 关键词

```text
十九世纪地理图志
铜版雕刻
博物馆档案
科学插图
```

### 特征

- 象牙白纸；
- 极细铜版线；
- 高密度 index contour；
- 文章点像测量站；
- 篇数用图志编号系统表达；
- 适合研究、出版和纸质打印。

与 `survey-parchment` 相比，它更文学、更出版；与 `mythic-parchment` 相比，它更理性、更少叙事。

---

## 推荐顺序

1. 保持 `survey-parchment` 为默认；
2. 先升级共享地形算法；
3. 再实现 `mythic-parchment`；
4. 两套皮肤共享同一语义数据和地形高度场，只替换线条、排印、纸张和控制组件；
5. 最后再考虑 `archive-engraving`。

皮肤不能拥有自己的山名、坐标或语义规则。否则同一份知识会因为换皮肤而变成不同地图。
