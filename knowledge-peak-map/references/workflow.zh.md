# 文山.skill 中文工作流

![文山.skill 黑白灰山群效果；页面可切换中文与 English](../assets/demo-bilingual.png)

**用山脉展示你的篇章。**

## 输入

- 作者昵称：用于页眉与传播图署名。
- 文章集合：一个 Obsidian 文件夹、Markdown 文件夹或明确文章清单。
- 界面语言：`zh`。

只分析用户选定的集合，不默认扫描整个 Vault。用户决定语料范围，但不预先定义山名。

## EGLFA 八步流水线

完整方法定义见 [EGLFA：证据门槛式纵向框架分析](methodology.zh.md)。

1. 确定作者、集合、时间范围、文档类型和作者归属。
2. 版本归并后，以一篇 canonical 文章作为独立分析单位。
3. 逐篇提取场景、行业、角色、实践、知识领域、claim、premises、evidence、日期和置信度。
4. 声明一个分类轴，主轴编码形成 MECE 候选主山；合并同义标签，把被父山完整包含的媒介、格式、方法或视角降为子峰，再审阅山与山之间的关系强度。
5. 至少三篇独立 canonical 文章通过证据门。
6. 用六个固定问题审查山界与父子包含关系，触发条件时必须人工复核。
7. 按时间综合早期、修正和稳定主张，形成当前回答。
8. 同一语料独立分析三次，通过稳定性门槛后再发布地图。

合格山名：

- 与实际场景或行业有强关系。
- 以名词或名词短语为主。
- 中文通常为 2–8 个字，必要时可保留行业缩写。
- 足够具体，读者知道文章大致在谈什么。
- 又足够宽，能够由至少三篇独立文章共同支撑。

例如：

| 当前文章证据 | 合格主标题 | 不适合作主标题的抽象回答 |
|---|---|---|
| AI模型、工具选择、工作流接入 | AI工具 | 摩擦决定采用 |
| 模型产业、访问权、人机责任边界 | AI行业 | 能力普及，判断升值 |
| 产品验证、需求判断、上线决策 | 产品经理 | 方向判断先于执行 |
| 数控加工、机床、刀具与工艺 | CNC | 制造精度来自约束 |
| 设备、传感器、硬件体验 | 智能硬件 | 体验取决于软硬协同 |
| 亲子关系、成长与家庭记录 | 育儿生活 | 陪伴无法被加速 |

右栏内容可以成为副标题或详情判断，但不能代替山名。

每座山至少由三篇独立 canonical 文章支撑。没有证据就没有山，不显示“待勘探”。

## 地图生成

主标题显示 Agent 识别的场景/行业关键词；副标题显示 Agent 对这批文章在该关键词下的回答或判断。篇数决定海拔和山体量。

山头数量不设硬上限。所有通过证据门并通过 MECE 审查的主山共同构成一个山群。父子包含关系用主山与子峰表达；主山之间再通过经过审阅的远近、连接与疏密组织。几千篇文章通过缩放、标签避让和点击详情组织，而不是为了版面删除有效主题。

篇数只代表写作积累量，不代表知识水平或判断正确性。地图右下角显示本次生成时间。

双语字段使用 `label`、`answer` 与经审阅的 `label_en`、`answer_en`。证据文章标题保持原文。

```bash
python3 scripts/render_territory_demo.py \
  --scope "/绝对路径/文章集合" \
  --nickname "作者昵称" \
  --language zh \
  --output-name "文山"
```

## 地图读法

- 主标题：从当前文章集合识别出的场景或行业关键词。
- 副标题：Agent 对该关键词下文章共同判断的回答。
- `13篇`：由 13 篇独立 canonical 文章形成的海拔。
- 点：文章证据，点击后返回原文。
- 等高线：该山内部的证据聚集形状。
- 跨山关系：可以作为桥显示，但不重复增加第二座山高度。
- 山间距离：主题共享的场景、实践、论证或演化关系越强，距离越近。
- 山脊与鞍部：相近山峰的高度场自然连接，不代表文章重复归属。

## 发布前校验

- 山名不是抽象规律、口号、金句、句子或用户愿望。
- 山名与真实场景、行业、角色或实践有强关系。
- 山名是 Agent 基于文章识别的，不是未经审阅地复制文件夹名。
- 主山共享一个分类轴并通过 MECE 审查；被父山包含的媒介、格式、方法或视角只作为子峰。
- 每座山至少三条唯一 canonical 源路径。
- 各山篇数之和等于地图独立文章数。
- 重要山间关系有审阅理由；无关系山峰不会为了版面被强行拉近。
- 不显示本地绝对路径、调试标签或虚构候选山。
- 13 寸屏可完整查看；1080×1440 截图不裁山、不带控制按钮。
