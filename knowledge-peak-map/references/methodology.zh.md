# EGLFA：证据门槛式纵向框架分析

## 定位声明

**证据门槛式纵向框架分析**（Evidence-Gated Longitudinal Framework Analysis，EGLFA）是文山为个人写作语料组合既有研究方法形成的方法规格，**不是已经存在的论文方法名**。

它最准确的用途是：

> 对个人写作语料进行证据门槛驱动的纵向主题分析与知识地图可视化。

理论来源：

- [Framework Method](https://doi.org/10.1186/1471-2288-13-117)：用 case × code 矩阵组织可审计的定性材料。
- [定性内容分析](https://pmc.ncbi.nlm.nih.gov/articles/PMC7932246/)：用系统编码形成类别、主题与模式。
- [扎根理论开放与主轴编码](https://link.springer.com/chapter/10.1007/978-3-030-15636-7_4)：从材料形成概念，再建立类别关系。
- [纵向定性分析](https://pmc.ncbi.nlm.nih.gov/articles/PMC8459825/)：比较主题、主张与经验如何随时间变化。
- [Argument Mining](https://aclanthology.org/J19-4006/)：识别 claim、premise、evidence 及其支持关系。
- [Reading Tea Leaves](https://papers.nips.cc/paper_files/paper/2009/hash/f92586a25bb3145facd64ab20fd554ff-Abstract.html)：统计表现更好的 topic model 不一定更具人类语义解释力，因此最终必须做人类可理解性验证。

文山不是 topic modeling，也不是 embedding 聚类。它以文章为分析单位，以框架矩阵为审计结构，以独立 canonical 文章为证据门槛，以论证综合提取作者目前形成的回答。

## 固定八步

### 1. 确定语料边界

记录：

- 作者；
- 文件集合；
- 时间范围；
- 文档类型；
- 是否为作者本人作品。

排除第三方参考、提示词、操作说明、模板、空文件、未完成碎片和非作者作品。

如果文档缺少可靠日期，或无法形成有意义的时间范围，不得宣称完成了“纵向分析”。应先修复日期，或把输出降级描述为“证据门槛式框架地图”，不生成早期、修正与稳定主张的时间结论。

### 2. 确定分析单位

一篇经过版本归并后的 canonical 文章是一个独立分析单位。草稿、成稿和改写不得重复增加海拔。

### 3. 逐篇开放编码

每篇至少提取：

```json
{
  "scene": [],
  "industry": [],
  "role": [],
  "practice": [],
  "knowledge_domain": [],
  "claim": "",
  "premises": [],
  "evidence": [],
  "date": "",
  "confidence": 0.0
}
```

### 4. 主轴编码形成候选山

先合并反复出现的具体领域，形成不限数量的候选子峰。候选山名必须：

- 是名词或名词短语；
- 与场景、行业、角色、实践或知识领域有强关系；
- 不是完整结论或金句；
- 不是未经审阅的文件夹名；
- 不是因为品牌重复出现而自动形成的供应商山。

候选山形成后，执行**山群关系审查**：

1. 声明一个主山分类轴，例如 `primary_problem_space`。
2. 主山必须在该轴上通过 MECE 审查；合并真正同义或重复的标签，但不要因为两个方向相关就强行归并。
3. 如果候选主题是另一个主题内部的媒介、格式、工具、方法或观察视角，将其保留为可审计子峰，不与父山平级。
4. 同一文章仍只能增加一座主山海拔，跨山关系不重复计数。
5. 对每一对重要山峰记录关系强度与理由，例如共同场景、共同实践、因果关系或时间演化。
6. 关系强度只决定地图中的远近与山脊连接，不决定文章是否属于该山。
7. 没有固定山头数量上限；所有通过证据门、MECE 审查且语义可解释的主山都可以进入同一山群。
8. 山头过多时通过画布缩放、标签避让与按层查看解决，不通过删除主题解决。

### 5. 通过证据门

一座山至少满足：

- 三篇独立 canonical 文章；
- 每篇都能解释归属理由；
- 副标题覆盖山内大多数文章；
- 同一文章只增加一座主山海拔；
- 没有证据就没有山。

### 6. 山界审查

固定回答六个问题：

1. 是否吸收了超过一半的有效文章？
2. 山内是否存在三个以上稳定子主题？
3. 任取五篇，是否都能解释为什么属于这座山？
4. 副标题是否覆盖至少 70% 的山内文章？
5. 如果拆山，子山是否仍各有三篇以上证据？
6. 该候选山是否只是另一座山内部的媒介、格式、方法、工具或观察视角？

如果前两个为“是”，第三至第五个有任何一个为“否”，或第六个为“是”，必须进入人工复核。审查结果必须写入 terrain，而不是只留在 Agent 的对话中。

山界审查之外，整张地图还必须通过山群关系审查：关系有原文或编码依据、远近可解释、无关系的山不会被强行拉近。单座山成立，不意味着它与其他山的空间关系自动成立。

### 7. 综合形成当前回答

副标题不得直接自由创作。按以下链路生成：

```text
提取每篇 claim
→ 合并同义主张
→ 找出反例与冲突
→ 按时间排列
→ 区分早期主张、修正主张、稳定主张
→ 形成当前回答
```

推荐句式：

> 在【领域】中，作者反复得出的回答是：【稳定判断】。

地图可以省略前半句，只显示稳定判断。

### 8. 稳定性验收

同一份语料独立分析三次，比较：

- 山峰数量差异 ≤ 1；
- 核心山名语义一致率 ≥ 80%；
- 文章主山归属 Jaccard ≥ 0.75；
- 海拔计数 100% 一致；
- 所有证据均可回到原文。

未达到门槛时，不得把结果描述为稳定知识地图；应标为待复核分析。

## 地图语义

- 篇数：长期积累量，不代表知识水平或观点正确性。
- 山名：作者长期反复书写的具体领域。
- 山群：所有通过证据门的山共同构成的连续知识地形。
- 山间距离：经过审阅的关系强弱；越近表示共享场景、实践、论证或演化关系越强。
- 山群疏密：相关山峰与证据在局部空间中的集中程度，不表示主题优劣。
- 副标题：作者截至当前在该领域形成的稳定回答。
- 时间地层：早期主张、修正主张与稳定主张的演化。
- 证据点：可回到原文的独立 canonical 文章。
- 等高线：由所有山峰的高度场合成为一张连续地形。相近山峰共享山脊与鞍部，远山自然分开；高分辨率曲线平滑保证丝滑。文章点只作为可回链证据，不代表 embedding 距离。
